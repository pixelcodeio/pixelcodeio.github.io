Features

Industry Swift Code
Our generated code follow industry standard iOS developer conventions and uses AutoLayout to layout the views so that your designs are pixel-perfect across screens of all different sizes.
Navigation
We also support in-app navigation between screens, getting rid of yet another step for iOS developers.
Comments
To improve the iOS development process even more, we implemented commenting on all uploaded designs which all team members can view and participate in.
Style Guide
We offer design style guides when you export your designs so that your developers can easily view all the fonts, sizes, and colors used.
Sketch Friendly
Upload Sketch artboards through the Pixelcode plugin that directly syncs onto the web application.
More Features
We currently support the core set of iOS components such as: UIButton, UILabel, UIImage, UITableView, UICollectionView, UINavBar and more!

Learn

List Views
These are defined by a group with [Name]ListView. There is an optional bound rectangle that defines the wash, and everything else must be inside groups called [Name]Cell
Tab Bars
Tab bars must be in a group with tabBar in its name. Active tab bar buttons must contain activeTabButton in its name. Nonactive must contain tabButton in its name.
Buttons
Buttons within a cell must be labeled [Name]Button
Labels (Text)
Text do not have to be specifically labeled! Just the [Name] is fine.
Images
Images, polygons, paths, circles do not have to be specifically labeled! Just the [Name] is fine.
